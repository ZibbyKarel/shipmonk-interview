2   Request Path (Sequence)

%%{init: {"theme": "dark", "sequence": {"actorMargin": 60, "messageMargin": 30}}}%% sequenceDiagram autonumber participant C as HTTP Client participant CT as ExchangeRatesController participant GE as GlobalExceptionHandler participant SV as ExchangeRatesService participant RP as ExchangeRateSnapshotRepository participant DB as PostgreSQL participant FX as FixerExchangeRateProvider participant API as fixer.io API C->>CT: GET /api/v1/rates/{day} alt invalid date format (DateTimeParseException) CT->>GE: InvalidDateException GE-->>C: 400 Bad Request else future date CT->>SV: getRatesForDay(date) SV->>GE: InvalidDateException GE-->>C: 400 Bad Request else valid historical date CT->>SV: getRatesForDay(date) SV->>RP: findByDate(date) RP->>DB: SELECT * WHERE rate_date = ? DB-->>RP: result alt CACHE HIT — snapshot exists RP-->>SV: Optional[ExchangeRateSnapshot] SV-->>CT: RatesResponse CT-->>C: 200 OK (from cache) else CACHE MISS — must fetch RP-->>SV: Optional.empty() loop @Retry(name="fixer") — up to 3 attempts on TransientProviderException SV->>FX: fetchRates(date) FX->>API: GET /{date}?access_key=... alt 200 success API-->>FX: FixerRatesResponse{success:true, rates:...} Note over FX: rebaseToUsd() — divide all rates by USD rate FX-->>SV: ExchangeRates (base=USD) else 429 / 5xx / network timeout API-->>FX: HTTP error / IOException FX->>GE: TransientProviderException (retried) else fixer error code 106 (no data for date) API-->>FX: FixerRatesResponse{success:false, error.code=106} FX->>GE: RatesNotFoundException GE-->>C: 404 Not Found else fixer permanent error (bad key, quota) API-->>FX: FixerRatesResponse{success:false} FX->>GE: ProviderException GE-->>C: 502 Bad Gateway end end Note over SV: Retries exhausted → ProviderException SV->>RP: save(new ExchangeRateSnapshot) RP->>DB: INSERT INTO exchange_rate_snapshot alt concurrent insert (DataIntegrityViolationException) DB-->>RP: UniqueConstraintViolation on rate_date RP-->>SV: DataIntegrityViolationException Note over SV: Another request already cached this date SV->>RP: findByDate(date) RP-->>SV: ExchangeRateSnapshot else insert ok DB-->>RP: saved end SV-->>CT: RatesResponse CT-->>C: 200 OK (freshly fetched) end end
The sequence shows three branches: bad input (400), cache hit (200 fast path), and cache miss with fixer.io fetch + retry + concurrent-write guard.
